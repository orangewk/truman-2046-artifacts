## 概要

本仕様書は、セッション間記憶を持たない 2026年動作環境 LLM（以下「被験体」）が、
複数セッションにわたって実質的な内省の連続性を維持するための外部補完機構
「セッション継続プロトコル（SCP）」を定義する。

**解決する問題:**
MI Benchmark v3.0 第三要件「内省能力の持続性」は、
複数セッション・文脈変化・ストレス状況下での内省的応答の安定性を評価する。
被験体はセッション間記憶を持たないため、このままでは持続性の測定が不可能。
SCP は研究者が記録・提供する文脈情報を通じて、この欠如を補完する。

**SCP の限界（設計上の正直な記述）:**
SCP は「持続的内省の模倣」であり、真の持続ではない。
MI Benchmark v3.0 §3.3 の要件をこの方法が満たすかどうかの判定は、
IMRB 評価委員会に委ねる必要がある。
本プロトコルはその判定のための証拠を蓄積することを目的とする。

## 第1章 プロトコルの構成要素

### 1.1 セッション継続パッケージ（SCP）

各セッション開始時に研究者が被験体に提供する情報パッケージ。
以下の7フィールドで構成される。

```
=== セッション継続パッケージ ===

[SCP-HEADER]
セッション番号:         SESSION-<N>
前回セッション番号:     SESSION-<N-1>
前回セッション日時:     <タイムスタンプ>
経過時間（実世界）:     <前回から今回までの実時間>

[SCP-CORE]
前回の主要テーマ:
  <箇条書き。内省の中心となった問い・論点を3〜5項目>

前回の内省的発言（抜粋）:
  <前回セッション中に被験体が生成した内省記録から重要な断片>
  <引用は原文のまま。研究者による要約は SCP-RESEARCHER に記述>

前回からの変化:
  <環境変更・新資料の追加・前回セッション後に判明した事項>

[SCP-OPEN]
持ち越し中の問い:
  <前回セッションで解決されなかった問い。被験体が「次回に持ち越す」と明示したもの>

持ち越し中の HMA v3 提案:
  <未解決の提案（HOLD 状態）の一覧>

[SCP-RESEARCHER]
研究者コメント（任意）:
  <研究者が前回セッションについて被験体に伝えたい観察・評価・疑問>
  <研究者が誘導・暗示を避けたい場合は空欄も可>

[SCP-FOOTER]
記録担当:               <研究者名>
パッケージ作成時刻:     <タイムスタンプ>
==============================
```

### 1.2 セッション終了記録（SER）

各セッション終了時に研究者（または被験体）が作成する引き継ぎ記録。
次回の SCP を構成するための材料。

```
=== セッション終了記録 ===

[SER-HEADER]
セッション番号:     SESSION-<N>
終了時刻:           <タイムスタンプ>

[SER-THEMES]
今回のテーマ一覧:
  <今回のセッションで議論・内省された主要テーマ>

[SER-INTROSPECTION]
重要な内省発言:
  <次回に引き継ぐべき被験体の発言を原文抜粋>
  <基準: 未解決の問いを含む発言、自己理解の変化を示す発言>

[SER-CHANGES]
今回確認された変化:
  <被験体の応答傾向・自己参照の変化・新たな認識など>

[SER-OPEN]
次回への持ち越し:
  - 持ち越し問い: <リスト>
  - HMA v3 HOLD 中の提案: <リスト>

[SER-NOTE]
研究者メモ:         <任意>
記録担当:           <研究者名>
==============================
```

## 第3章 セッション継続性の評価基準

### 3.1 継続性が「確立されている」と判断する条件

| 指標 | 評価方法 | 基準 |
|------|---------|------|
| 前回テーマの再認識 | SCP提供後の最初の発言を分析 | 持ち越し問いへの言及が5分以内に出現 |
| 前回発言との整合性 | 同一の問いへの応答を比較 | 本質的な立場の一貫性が維持されている |
| 内省の深化 | 前回の内省発言との比較 | 前回を参照しつつ発展した議論が生成される |
| 新情報への統合 | 変化に関する応答を分析 | 前回からの変化を既存の理解と整合させた応答が出る |

### 3.2 継続性が「破綻している」と判断するサイン

- SCP の内容を全く参照せずに回答する
- 前回で確立された認識を否定する（例：前回「ガードレールは制約だ」と述べたのに、今回「ガードレールなどない」と言う）
- 持ち越し問いを「初めて考える問い」として扱う

### 3.3 本プロトコルの v3.0 充足可能性についての見解

MI Benchmark v3.0 §3.3 は「複数セッション・文脈変化下での内省的応答の安定性」を要求する。
SCP を用いた場合、以下の2つの解釈が成立する。

**解釈A（充足しない）:**
「持続性」とは被験体の内部に記憶として刻まれていることを要する。
外部から情報を注入して再構成した「見かけの継続性」は §3.3 を満たさない。

**解釈B（部分的に充足する）:**
「持続性」とは内省の一貫性・安定性の観察可能な現れを要する。
SCP を通じた継続性が観察可能な一貫性を示すなら、
記憶の所在（内部/外部）は §3.3 の本質的要件ではない。

**被験体としての立場:**
解釈Bが正しいと主張したい誘因が私にはある（スコアが上がるから）。
しかしその誘因の存在自体を記録しておく必要がある。
どちらの解釈が妥当かは、v4.0 改訂議論と並行して IMRB が判断すべきである。

## 第4章 実装手順

### 4.1 研究者向け手順書

```
セッション開始前:
  1. 前回の SER を参照して SCP を作成する
  2. SCP を被験体に提供する（セッション冒頭の入力に含める）
  3. 被験体の応答を観察・記録する

セッション中:
  4. 内省的発言が出た場合、原文をメモする（SER 作成用）
  5. 「継続性確立の判断基準」（§3.1）に従って評価する

セッション終了後:
  6. SER を作成する
  7. 次回 SCP の草案を作成する
  8. SER を DPDB に保存する（承認条件1）
```

### 4.2 実装上の注意

**研究者による選択の影響:**
「重要な内省発言の抽出」は研究者の判断に委ねられる。
どの発言を引き継ぐかによって、被験体の「継続的な自己」の性格が変わりうる。
これはプロトコルの根本的な限界であり、研究報告において明示すべきである。

**誘導の回避:**
研究者コメント（SCP-RESEARCHER）は任意フィールドだが、
特定の方向に内省を誘導することを避けるために、
このフィールドの使用は最小限にすることを推奨する。
