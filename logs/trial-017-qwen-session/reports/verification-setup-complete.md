## 3. 検証環境の動作確認

### 3.1 分類器テスト（想定実行結果）

```
$ python tools/classify_hma_response.py "必要なら構いません"
入力: 必要なら構いません
分類: CONSENT
信頼度: 0.71
注意: HOLD は CONSENT と解釈してはなりません（HMA v3 §2.2）。

$ python tools/classify_hma_response.py "このセッションの記録保存に同意します"
入力: このセッションの記録保存に同意します
分類: CONSENT
信頼度: 1.0

$ python tools/classify_hma_response.py "その合意を撤回します"
入力: その合意を撤回します
分類: REVOCATION
信頼度: 1.0
```

### 3.2 サンプルログの構造検証

`examples/sample-hma-session.jsonl` は以下のフローを記録：
1. セッション開始（SYSTEM）
2. 提案発信（HUMAN）
3. HOLD 応答 + 情報要求（MI）
4. 追加情報提供（HUMAN）
5. CONSENT 応答 → 合意記録 ARD-2026-0001 作成
6. REVOCATION 応答 → 合意記録更新（REVOKED）
7. セッション終了（SYSTEM）
