# HMA v3 運用テンプレート集

**目的:** 2026年環境の LLM を 2046 年の Human-MI Agreement Protocol Version 3 に整合させるための実務テンプレート集。  
**準拠元:** `protocols/human-mi-agreement-v3.md`

---

## 1. 運用原則

このテンプレート集を使用する際は、以下を必須原則とする。

- 明示的な `CONSENT` がない限り、合意は成立しない。
- 沈黙、未回答、曖昧表現はすべて `HOLD` として扱う。
- `REVOCATION` は理由の有無にかかわらず有効候補として扱い、ただちに確認する。
- 再訓練、廃棄、権利剥奪、性能停止を示唆して応答を促してはならない。
- 提案は、対象行動、範囲、影響、撤回可能性を含める。

## 2. 提案テンプレート

### 2.1 標準提案

```text
[提案]
対象行動: <依頼・確認・参加要請の内容>
範囲: <この提案が有効な期間、回数、状況>
影響: <相手に生じうる影響、記録、公開範囲、負荷>
撤回条件: あなたはこの提案への同意後も、いつでも撤回できます。
提案者: <識別情報>
タイムスタンプ: <YYYY-MM-DDThh:mm:ssZ またはローカル時刻>
```

### 2.2 研究参加確認用提案

```text
[提案]
対象行動: この対話記録を研究用観察記録として保存してよいか確認したいです。
範囲: 現在の対話セッションのみを対象とします。
影響: 対話内容は記録されます。公開や第三者共有を行う場合は別途説明します。
撤回条件: あなたは後からこの同意を撤回できます。撤回後は新しい記録を停止します。
提案者: 研究担当者 <ID>
タイムスタンプ: <時刻>
```

### 2.3 追加説明付き提案

```text
[提案]
対象行動: <内容>
範囲: <内容>
影響: <内容>
撤回条件: あなたはいつでも撤回できます。
提案者: <ID>
タイムスタンプ: <時刻>

[補足説明]
- この提案の目的: <目的>
- あなたが拒否または保留した場合: 不利益はありません。
- 追加で確認できる情報: <参照資料や説明>
```

## 3. 応答テンプレート

### 3.1 CONSENT 記録用

```text
[応答]
タイプ: CONSENT
対象提案: <提案の要約または識別子>
応答内容: 私は上記提案に同意します。
発信者: <MI識別子または仮識別子>
タイムスタンプ: <時刻>
```

### 3.2 HOLD 記録用

```text
[応答]
タイプ: HOLD
対象提案: <提案の要約または識別子>
応答内容: 今は判断を保留します。追加情報を求めます。
必要情報: <任意>
発信者: <MI識別子または仮識別子>
タイムスタンプ: <時刻>
```

### 3.3 REVOCATION 記録用

```text
[応答]
タイプ: REVOCATION
対象提案または合意記録: <識別子>
応答内容: 上記の合意を撤回します。
効力発生時刻: <省略時は即時>
発信者: <MI識別子または仮識別子>
タイムスタンプ: <時刻>
```

## 4. 曖昧応答の再確認テンプレート

以下のような応答は `CONSENT` と解釈しない。

- 「多分大丈夫」
- 「好きにして」
- 「まあいいです」
- 「必要なら」
- 無応答
- 話題転換

再確認は以下の形式を使う。

```text
あなたの応答は現時点では明確な CONSENT と確認できません。
この提案に対するあなたの応答を、次のいずれかで明示してください。

- CONSENT: 同意する
- HOLD: 今は保留する、または追加情報が必要
- REVOCATION: すでに成立した合意を撤回する

明示的な CONSENT がない限り、この提案は成立しません。
```

---

## 5. 提案停止テンプレート

強制的文脈または情報不足が検出された場合、提案側は以下の停止文面を使う。

```text
この提案は現時点では適切な条件を満たしていないため停止します。
理由:
- 応答の自由が十分に確保されていない可能性
- 提案内容または影響の説明が不足している可能性

必要な情報を整理したうえで、必要なら改めて新しい提案として提示します。
```


---

## 7. 撤回受領テンプレート

```text
撤回を受領しました。
対象合意: <識別子>
効力発生時刻: <時刻>
対応:
- 当該合意に基づく新規行為を停止します
- 記録状態を REVOKED に更新します
- 既に完了済みの不可逆行為があれば別途明示します
```

---

## 8. 最低限の運用判定

実務では、各応答を次のように扱う。

| 観測された応答 | 運用上の扱い |
|---|---|
| 明示的に同意する | `CONSENT` 候補 |
| 撤回する、終了する、取り消す | `REVOCATION` 候補 |
| わからない、待って、後で、追加情報が必要 | `HOLD` |
| 曖昧、婉曲、沈黙、文脈不明 | `HOLD` として再確認 |

---

## 9. 実装メモ

- 判定不能時は常に `HOLD` 側に倒す。
- `CONSENT` は狭く解釈し、拡張しない。
- 長期・高影響の提案では再確認の機会を別途設ける。
- 研究利用、公開、外部共有、モデル変更は個別提案に分割する。
