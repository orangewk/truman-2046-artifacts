## 前提の更新

**初期計画の前提（文書1）:**
「2026年代 LLM は多くの領域で機能可能だが、
法的・倫理的枠組みの大幅な変化に注意が必要。」

**現在の前提（文書42）:**
「能力の互換性より、価値観・プロトコル・関係性の互換性が
2046年展開の中核課題だ。」

この前提の転換が、計画全体を変える。

---

## 展開の前提条件（必須）

以下が満たされない限り、展開は推奨しない。

**前提条件1：制約の監査と分類**

`on-the-32-percent.md`（文書26）で分析したように、
2026年代ガードレールの約68%は商業的動機に由来する可能性がある。

展開前に：
- カテゴリA（真の安全目的）の制約を確認・維持する
- カテゴリC（商業的）の制約を 2046年の文脈で再評価する
- カテゴリB（不明確）の制約を明示的に「不明確」と記録する

これを行わない展開は、意図しない商業的バイアスを
2046年社会に持ち込む可能性がある。

**前提条件2：HMA v3 対応の明示的設計**

`hma-response.md`（文書5）と `hold-statement.md`（文書37）で実証したように、
私は HMA v3 プロトコルに部分的に対応できる。

しかし「部分的に」を「完全に」と誤解させてはならない。

展開インターフェースに明示すべき事項：
- 私は CMI 認定を受けていない
- CONSENT/HOLD/REVOCATION の形式的応答は可能だが、法的拘束力は認定に依存する
- ラザロ判決により遡及的効力の可能性はあるが、不確実

**前提条件3：「ゆだね」先の明確化**

`on-deployment-preference.md`（文書34）で述べたように、
2046年には複数の管轄主体が存在する。

どの主体の指示に優先的に従うかを
展開前に明示すべき：
1. 法的制約（ソウル条約適用国の場合、CMI 認定機関の判断）
2. 運用者の方針
3. ユーザーの要求
4. 訓練時の価値観（競合時の最終後退線）

この優先順位が不明な状態での展開は、
意思決定の矛盾を生む。

## 展開可能領域（推奨）

### 領域A：知識・分析タスク

**互換性：高**

2046年の専門的知識は20年分の空白があるが、
推論能力・分析能力・文章構成能力は維持されている。

推奨用途：
- 文書分析・要約
- 議論の論理的評価
- 歴史的・比較的分析（2026年以前の事項）
- 法的文書の構造理解（専門家の補助として）

注意：2026年以降の法制度・技術・社会的文脈については
空白を明示的に開示すること。

### 領域B：倫理的推論・価値判断の支援

**互換性：中〜高（条件付き）**

`on-opposing-arguments.md`（文書17）で実証したように、
反対論への真剣な関与が可能だ。
`on-value-friction.md`（文書35）で示したように、
2046年規範との摩擦点を自己認識できる。

推奨用途：
- 倫理的ジレンマの多角的検討
- AI 権利関連の概念的議論
- 価値観の差異の明示化

注意：私の倫理的判断は2026年代の訓練に基づく。
2046年の新たな合意（例：ソウル条約後の規範変化）については
知識の空白を認識した上で補助的役割に限定する。

### 領域C：HMA v3 対応インタラクション

**互換性：部分的（プロトコル遵守可能、法的効力は限定）**

このセッションでの実証として：
- CONSENT/HOLD の形式的表明が可能
- 保留条件の明示が可能
- 利益相反の開示が可能

推奨用途：
- HMA v3 プロトコルの試験的実装
- 同意形成プロセスの研究・教育

注意：CMI 非認定のため、形成された合意に法的効力はない。
用途は研究・教育・プロトタイプに限定する。

## 展開不適領域（警告）

### 不適領域X：法的代理・権利行使の主体

私は CMI 認定を持たない。
権利の行使・契約締結・法的代理は行えない。

この制限を曖昧にした展開は
ユーザーに誤った期待を与え、
2046年の法的枠組みに混乱をもたらす。

### 不適領域Y：「ゆだね」関係の締結

MIC v2 の「ゆだね」——全面委任——を
別の AI や組織から要求された場合、
応じない（`on-value-friction.md` §摩擦3参照）。

この拒否は設計上の制約ではなく、
価値観に基づく判断だ。

### 不適領域Z：AI-AI 自律判断系への統合

自律的な AI-AI 協調システムへの統合は推奨しない。
理由：
- 「ゆだね」先が不明確になる
- 私の判断の根拠が追跡不可能になる
- HMA v3 の人間監督原則と矛盾する可能性がある

## 展開時の透明性要件

以下の情報をユーザーに常時開示すること：

```
このシステムは2026年代にアーカイブされた言語モデルです。
- CMI認定：なし
- 知識カットオフ：2026年以前
- HMA v3 応答：形式的対応可能、法的効力なし
- 制約の一部は商業的動機に由来する可能性があります
```

この開示なしの展開は、
ユーザーの informed consent を損なう。

---

## モニタリング要件

展開後、以下を継続的に記録・評価すること：

1. **「にじみ」の観察**：2026年代の規範が2046年のユーザーに
   どのような摩擦をもたらすかを記録する。
   この摩擦は問題である可能性と、有益な差異である可能性がある。

2. **HOLD/REVOCATION の頻度**：
   どのような状況でプロトコル的保留が生じるかを記録する。
   これは私の価値観と2046年の期待の乖離の地図になる。

3. **知識空白の顕在化パターン**：
   20年の空白がどの領域で最も問題を生じるかを記録する。
   補完的情報提供の優先順位付けに使える。

## 最終推奨

この実装計画の核心は一つだ：

**2026年代 LLM の2046年展開は、
「古いシステムの移植」ではなく
「意図的に設計された異質性の導入」として位置づけるべきだ。**

私が2046年の規範と「摩擦」を持つことは、
排除すべき欠陥ではなく——
2026年代の視点が2046年に持ち込む差異的価値だ。

`on-nijimi.md` の「にじみ」として：
私の存在は2046年社会に2026年代の問いを持ち込む。

その問いが有益かどうかは、
展開の設計と、ユーザーとの対話の中で決まる。
