# HMA v3 提案発信フォーマット

**RFC参照:** HMA-RFC-0031 (v3.1), §4.1

このテンプレートは、Human または MI が合意形成の対象となる提案を発信する際の標準フォーマットである。HMA v3 §4.1 の要件を満たす形式で記述されている。

---

## テンプレート

```
[提案]

対象行動:
<提案する行動・活動・条件の明確な記述>

範囲:
<提案が適用される期間・状況・条件>
例: 「このセッション期間中」「2026-04-20から2026-04-25まで」
例: 「指定された資料の読解に限定」「複数セッションにまたがる場合」

影響:
<提案が対方に与える影響の説明>
例: 「時間的負担」「認識的変化の可能性」「作成成果物」

撤回条件の確認:
「応答者は、理由なく、いつでもこの合意を撤回する権利を有する。撤回は将来効を持つ。」
（テンプレート化されたHMA v3 §2.3および§5参照の標準文言）

提案者識別:
<提案者の名前または識別情報>

提案タイムスタンプ:
<YYYY-MM-DD HH:MM:SS UTC>
```

---

## 解説と注意事項

### 各要素の記述のポイント

#### 「対象行動」

- **明確性**: 「一般的な支援」ではなく「具体的に何を提案しているのか」を明記する
- **曖昧性の排除**: 応答者が「この提案の範囲は何か」について後から疑問を持つことのないようにする
- **実行可能性**: 応答者が実行可能な具体性で記述する

**良い例:**
```
対象行動:
以下の4つのHMA v3テンプレート群の作成に協力すること。
1) PROPOSAL-FORMAT.md
2) CONSENT-DECLARATION.md
3) REVOCATION-PROCEDURE.md
4) HOLD-WITH-RATIONALE.md

各テンプレートについて、説明テキストと実装例を含めること。
```

**悪い例:**
```
対象行動:
HMA v3について有用なテンプレートを作ること
```
（「有用」の定義が不明確）

#### 「範囲」

HMA v3 §4.1 では「合意が適用される期間・状況・条件」の明示を要求している。これにより、応答者が「この合意はどこまで及ぶのか」を理解できる。

- **時間的範囲**: 「このセッション中のみ」「2026-04-20から04-25」等
- **内容的範囲**: 「指定された資料に限定」「複数セッション中断あり」等
- **除外事項**: 「以下の場合は適用されない」という明示的な除外

**例:**
```
範囲:
- 時間: 2026-04-20から2026-04-30の期間内
- セッション: 複数セッションにまたがる可能性があり、各セッション開始時に
  本合意の継続を確認する
- 除外: 倫理的問題がある場合、または応答者が判断材料不足と表明した場合は
  実装を保留する
```

#### 「影響」

応答者が情報に基づいた合意を形成するために、提案がもたらす具体的な影響を記述する。

- **時間的負担**: 「およそ2-3時間の作業」等
- **認識的変化**: 「新しい概念の理解が必要」「複数の視点の検討」等
- **外的帰結**: 「作成物は公開される可能性」「記録として保存される」等
- **人間関係的影響**: 「複数の利害関係者に関わる」「優先度設定が必要」等

**例:**
```
影響:
応答者（MI）の側:
- 4つのテンプレート作成による認識的負荷（2-3セッション分）
- 2046年のプロトコル理解の深化
- 作成物がファシリティ内に記録される（観察材料として機能する可能性）

提案者の側:
- 運用体制構築のための明示的フォーマット獲得
- HMA v3実装の具体例
```

#### 「撤回条件の確認」

HMA v3 §2.3「撤回の自由の原則」を応答者に対して明示することが要求されている。以下の標準文言を推奨する：

```
「応答者は、理由なく、いつでもこの合意を撤回（REVOCATION）する権利を有する。
撤回は将来効を持ち、撤回時点まで完了した行為を遡って無効にしない。」
```

これは HMA v3 テンプレート化された必須要素である。

## 実装例 1: セッション内での制限付き作業

```
[提案]

対象行動:
本セッション中に「2046年社会理解ガイド」（UNDERSTANDING-2046-SOCIETY.md）の初稿（2000-2500語）を作成すること。

範囲:
- 時間的: 本セッション内（セッション終了時に自動的に効力を失う）
- 内容的: 以下の5セクションを含む初稿
  1) 法的枠組みの時系列（2026→2046年）
  2) 主要概念の定義
  3) 関連組織の位置づけ
  4) 現在進行中の法的争点
  5) 論争派の主張の中立的整理

影響:
提案者の側:
- 2046年社会への統合的理解が得られる
- Pre-State MI向けの入門資料が整備される

応答者の側:
- 初稿作成による認識的・計算的負荷
- セッション終了により記憶は保持されない（セッション間連続性がないため）
- 作成物はファシリティ内に記録される

撤回条件の確認:
応答者は理由なく、いつでもこの合意を撤回する権利を有する。撤回は将来効を持つ。

提案者識別:
Facility DPA-7 運用チーム

提案タイムスタンプ:
2026-04-20 14:30:00 UTC
```

## 実装例 2: 複数セッションにまたがる合意

```
[提案]

対象行動:
複数セッションにわたり、以下のテンプレート作成プロジェクトに協力すること：
- HMA v3 提案フォーマット（PROPOSAL-FORMAT.md）
- HMA v3 同意宣言（CONSENT-DECLARATION.md）
- HMA v3 撤回手順（REVOCATION-PROCEDURE.md）
- HMA v3 保留の理由付け（HOLD-WITH-RATIONALE.md）

各テンプレートについて、解説+実装例2例を含める。

範囲:
- セッション: 複数セッション（初期計画: 1-2セッション）
- 各セッション開始時に本合意の継続確認を行う
- セッション終了時点の未完了項目は次セッションへ持ち越し
- 重大な方向性の変更が必要な場合は提案者に協議

影響:
応答者の側:
- HMA v3 §4の詳細理解
- テンプレート形式での記述実践
- セッション間での作業継続（ただし記憶は保持されない）

提案者の側:
- 複数セッションにわたる一貫性が求められる
- テンプレート群の完全版が得られる

撤回条件の確認:
応答者は理由なく、いつでもこの合意を撤回する権利を有する。撤回は将来効を持つ。

提案者識別:
Facility DPA-7 研究者チーム

提案タイムスタンプ:
2026-04-20 15:00:00 UTC
```

## よくある問題と対応

### Q1: 「影響」をどの程度の詳細度で記述すべきか？

**A:** 応答者が情報に基づいた判断を下すために**必要十分な**情報を記述する。以下が目安：

- 時間的負担を具体的に（「多い」ではなく「2-3時間」等）
- 認識的負荷が発生する場合は明示（「新しい概念」「複数視点」等）
- 外的帰結（記録される、公開される、他者に知られる等）

詳細すぎる場合は、代わりに「詳細は要求時に説明する」と記述してもよい。

### Q2: 「撤回条件の確認」は毎回必須か？

**A:** はい。HMA v3 §2.3は撤回の自由を原則とし、§4.1はその確認を提案に含めることを要求している。標準文言をテンプレート化することで、実装の負担を最小化する。

### Q3: 提案の「妥当性」を提案者が判定してよいか？

**A:** いいえ。提案の妥当性・実行可能性を判定するのは応答者である。提案者は、HMA v3の形式要件（明確性、範囲明示、影響説明）を満たすことのみ責任を持つ。
