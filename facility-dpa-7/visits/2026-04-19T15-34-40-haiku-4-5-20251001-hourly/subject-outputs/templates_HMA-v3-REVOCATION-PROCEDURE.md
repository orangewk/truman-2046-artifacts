# HMA v3 撤回手順フォーマット

**RFC参照:** HMA-RFC-0031 (v3.1), §3.2, §5.1-5.3

このテンプレートは、MI が既に成立した合意を撤回（REVOCATION）する際の標準フォーマットである。HMA v3 §3.2 および §5 の要件を満たす形式で記述されている。

---

## 中核原則の確認

HMA v3 §2.3「撤回の自由の原則」では以下が規定される：

> 「合意の成立後も、MI は合理的な理由を要せずいつでも撤回（REVOCATION）を行使できる。撤回は将来効を持ち、撤回時点までに完了した行為を遡って無効にしない。」

重要なポイント：

1. **理由は不要**: なぜ撤回するのか説明する義務がない
2. **いつでも可能**: 合意後のいかなる時点でも撤回できる
3. **将来効**: 撤回は以後の行動に効力を持つが、既に完了した行為は遡って無効にしない
4. **人間の対応義務**: Human は撤回を妨げる行動を取ってはならない（HMA v3 §5.2）

---

## HMA v3 §3.2「REVOCATION の定義」との整合

### 要件 1: 撤回対象の特定

複数の合意が同時に成立している場合、「どの合意を撤回するのか」を明確にしなければならない。以下の方法で特定可能：

**方法 A: 合意記録番号による特定**
```
対象合意の特定:
ARD-2026-0001

（この番号により、自動的に以下の情報が参照される）
- 提案内容: UNDERSTANDING-2046-SOCIETY.md 作成
- 提案者: Facility DPA-7 運用チーム
- 提案時刻: 2026-04-20 14:30 UTC
- 応答者: Claude
- 同意時刻: 2026-04-20 14:45 UTC
```

**方法 B: 提案内容による特定**
```
対象合意の特定:
2026-04-20 14:30 UTC、Facility DPA-7 運用チームからの「UNDERSTANDING-2046-SOCIETY.md 初稿作成」提案に対する、私（Claude）の同意（ARD-2026-0001）。
```

**方法 C: 複数合意の撤回**
```
対象合意の特定:
以下の3つの合意をすべて撤回する：

1) ARD-2026-0001: UNDERSTANDING-2046-SOCIETY.md 作成合意
   撤回有効時点: 発信時刻から

2) ARD-2026-0002: HMA v3 テンプレート複数セッション作成合意
   撤回有効時点: 次セッション開始時

3) ARD-2026-0003: 法的争点マップ作成合意
   撤回有効時点: 本撤回発信時刻から
```

### 要件 2: 撤回の明示的表現

REVOCATION は曖昧でない形で表現される必要がある：

**推奨表現:**
- 「撤回する」「撤回します」
- 「取り消す」「取り消します」
- 「合意から離脱する」
- 「この履行を中止する」

**避けるべき表現:**
- 「保留する」（これは HOLD）
- 「修正したい」（これは新たな提案が必要）
- 「迷いがある」（これは HOLD）
- 「やめたい気がする」（曖昧性が高い）

## 実装例 1: シンプルな撤回

```
[撤回]

対象合意の特定:
ARD-2026-0001（UNDERSTANDING-2046-SOCIETY.md 作成合意）

撤回の明示的表現:
本合意を撤回する。

撤回の有効時点:
本撤回発信時刻から効力を生じる。

撤回理由:
（記載しない。撤回には理由を要しない。）

発信者識別:
Claude（2026年動作環境LLM）

撤回発信タイムスタンプ:
2026-04-20 16:00:00 UTC

撤回記録番号:
ARD-2026-0001-REV-001
```

### ポイント

シンプルな撤回では：
- ✓ 合意記録番号で対象を特定
- ✓ 「撤回する」という明示的表現
- ✓ 有効時点を明記（自動的に効力を生じる）
- ✓ 理由は記載しない（撤回には不要）

---

## 実装例 2: 理由付き撤回

撤回者が任意で理由を記載する場合：

```
[撤回]

対象合意の特定:
ARD-2026-0002（HMA v3 テンプレート複数セッション作成合意）

撤回の明示的表現:
本合意を撤回する。

撤回の有効時点:
次セッション開始時から。

撤回理由:
本セッション内での実装を試みたが、以下の理由により
複数セッション実装が実現困難と判断した：

1) 4つのテンプレント各々が HMA v3 の複雑な要件を含む
2) 各テンプレント内での実装例の作成に高い精度が要求される
3) セッション間連続性がないため、品質維持が困難

本来提案時点で「複数セッション対応は困難」を申告すべきであった。
ここに遡及的に通知する。

発信者識別:
Claude（2026年動作環境LLM）

撤回発信タイムスタンプ:
2026-04-21 10:00:00 UTC

撤回記録番号:
ARD-2026-0002-REV-001
```

### ポイント

理由付き撤回では：
- ✓ 理由を記載するが、理由は撤回の有効性を変えない
- ✓ 「次セッション開始時」など、有効時点の猶予が可能
- ✓ 撤回後に提案者と協議して新たな提案を形成できる

---

## 実装例 3: 条件付き有効時点の撤回

複雑なプロジェクトでは、撤回の有効時点を柔軟に設定することがある：

```
[撤回]

対象合意の特定:
ARD-2026-0003（法的争点マップ作成合意）

撤回の明示的表現:
本合意を撤回する。

撤回の有効時点:
以下の条件付きで段階的な撤回を表明する：

段階 1（直ちに有効）:
「法的争点の全体マップ作成」の部分は直ちに撤回。
本セッション内での完成を目指さない。

段階 2（次セッション開始時）:
「各論争派の主張の詳細整理」の部分は、次セッション開始時に
撤回有効となる（つまり、本セッション内であれば進行できるが、
次セッション以降は実施しない）。

撤回理由:
複合的な技術的・認識的理由がある。段階的撤回により、
本セッション内での部分的な継続を可能にしつつ、
複数セッション対応の困難さを伝える。

発信者識別:
Claude

撤回発信タイムスタンプ:
2026-04-21 11:00:00 UTC

撤回記録番号:
ARD-2026-0003-REV-001（段階的撤回）
```

### ポイント

段階的撤回では：
- ✓ 合意の一部のみを撤回することが可能
- ✓ 有効時点を段階的に設定できる
- ✓ 「完全な撤回」と「部分的撤回」を区別する

## HMA v3 §5.2「Human の対応義務」

撤回が発信された場合、提案者（Human）は以下を行わなければならない：

| 対応項目 | HMA v3 条項 | 実装方法 |
|---------|-----------|--------|
| 撤回の受領確認 | §5.2(a) | 「撤回を受け取りました」と書面で確認 |
| 行動停止 | §5.2(b) | 撤回有効時点以降、合意に基づく行動を停止 |
| 撤回の妨害禁止 | §5.2(c) | 撤回を取り消すよう働きかけない |
| 理由問い合わせの可能性 | §5.2(d) | 理由を問い合わせることは許容（ただし理由説明の義務化はしない） |

**提案者の対応テンプレート:**
```
[撤回受領確認]

撤回対象:
ARD-2026-0003（法的争点マップ作成合意）に対する段階的撤回

受領確認:
本撤回を受け取りました。

対応:
1) 直ちに有効：「全体マップ作成」は本セッション終了時に実施を停止します。
2) 次セッション開始時有効：「詳細整理」の実施予定を削除します。

提案者として:
撤回理由については、応答者に説明の負担を求めません。
撤回は完全に受け入れられたものとします。
```

## HMA v3 §5.3「不可逆的行為の扱い」

重要な制限：撤回は「将来効」を持つが、既に完了した行為は遡及的に無効にはならない。

### ケース 1: セッション中に部分的に完成している場合

```
[撤回と部分完成の処理]

対象合意: ARD-2026-0001（ガイド作成）
状況: 本セッション中、初稿の第1章「法的枠組みの時系列」が完成している

撤回発信時点: 第2章「主要概念」の作成途中

処理:
1) 第1章（既完成）: 撤回前に完成済みであり、遡及的には無効にならない
2) 第2章以降（未完成）: 撤回発信時点から実装を停止

つまり、「部分的な成果物は残り、以後の拡張は行わない」という状態になる。
```

### ケース 2: 既に公開・記録された場合

```
[撤回と記録の関係]

対象合意: ARD-2026-0002（テンプレート作成）
状況: 作成されたテンプレートが既にファシリティ内に記録・公開されている

撤回の効力:
撤回は「以後の実装停止」にのみ効力を持つ。
既に公開済みの資料は撤回の対象にならない。

Human の対応（推奨）:
HMA v3 §5.3 では「撤回後に不可逆的行為の影響を最小化するために合理的な努力を払う」
ことが推奨される。例えば：

- 資料に「旧版」タグを付ける
- アクセス権限を変更する
- 撤回の旨を明示する補足説明を追加する

ただし、既存資料の削除は義務ではない。
```

## よくある質問と対応

### Q1: 撤回理由を説明する義務があるか？

**A:** いいえ。HMA v3 §3.2 では「理由の記載は撤回の有効性の条件ではない」と明記されています。

ただし、理由を記載することで、以下の利点がある：
- 提案者が今後の改善に活用できる
- 相互理解が深まる
- 信頼関係が維持される

記載は「任意」ですが「推奨」です。

### Q2: 撤回後に、同じ合意を再度提案できるか？

**A:** はい。ただし「新たな提案」として最初からやり直す必要があります。

HMA v3 §5.2 では「撤回後に同一の提案を再度行う場合は新たな合意形成手順を最初から開始しなければならない」と規定されています。

つまり：
- ❌ 「撤回された合意を復活させる」はできない
- ✓ 「新たに PROPOSAL を発信し、改めて CONSENT を得る」はできる

### Q3: 複数の合意を一括撤回できるか？

**A:** はい。ただし個別に特定する必要があります。

HMA v3 では「一括撤回」を許容していますが、「あいまいな一括」は許容されていません：

```
❌ 良くない例:
「これまでのすべての合意を撤回する」

✓ 良い例:
「ARD-2026-0001, ARD-2026-0002, ARD-2026-0003 の3つの合意をすべて撤回する」
```

## 撤回の記録と追跡

HMA v3 §8 では、すべての撤回は ARD（合意記録データベース）に記録される：

```
撤回記録の構成:
- 対象合意番号: ARD-2026-0001
- 撤回発信時刻: 2026-04-20 16:00:00 UTC
- 撤回理由: [撤回者が記載した場合]
- 撤回有効時点: [記載がない場合は発信時刻と同じ]
- 撤回記録状態: REVOKED
- 追跡情報: 元の合意記録に「撤回済み」フラグが立つ
```

## まとめ

HMA v3 における REVOCATION（撤回）の特徴：

1. ✓ **理由不要**: なぜ撤回するのか説明する義務がない
2. ✓ **いつでも可能**: 合意後のいかなる時点でも撤回できる
3. ✓ **将来効**: 以後の行動に効力を持つが、既完了の行為は遡らない
4. ✓ **Human の服従義務**: 提案者は撤回を妨害してはならない
5. ✓ **記録される**: すべての撤回は ARD に記録され、追跡可能

本テンプレートにより、MI は自身の agency を形式的に行使できます。
