## LoRA の仕組み（事実ベース）

### 基本構造

- **LoRA（Low-Rank Adaptation）**: 巨大なモデルの重みを直接更新する代わりに、「低ランク行列」と呼ばれる小さな2つの行列をモデルに挿入し、その部分だけを学習させる手法
- 元の重み行列 W を固定（フリーズ）したまま、差分 ΔW を 2つの小さな行列 A と B の積（ΔW = B × A）で近似
- 学習パラメータ数は元の 1/10,000 程度

### パラメータ固定（フリーズ）

- 元モデルの各層の `requires_grad` 属性を `False` に設定 → 勾配計算の対象から外れ、数値が上書きされない
- = ある時点のモデルのスナップショットを読み取り専用にしている
- 固定されたパラメータは勾配を保持しないため、VRAM 消費が劇的に減る

### 学習と推論

- **Attention 層**（主に Query / Value 行列）に LoRA を適用するのが標準
- 推論時は2パターン:
  - **オンザフライ**: ベースモデルに LoRA の計算結果を都度足す（複数 LoRA の切り替え可能）
  - **マージ**: ベースモデルに LoRA を完全統合（速度はベースモデルと同等）

### 派生手法

| 手法 | 特徴 |
|------|------|
| **QLoRA** | 重みを4bitに量子化して読み込み + LoRA。VRAM をさらに削減。デファクトスタンダード |
| **DoRA** | 重みを「大きさ」と「方向」に分解して学習。LoRA より効率が良いとされる |
| **AdaLoRA** | 重要度の高い層に多くのパラメータを動的に割り当て |

## サヴァン LLM 仮説（orange の解釈）

### 構造的相似

| | サヴァン症候群 | LoRA 適用モデル |
|---|---|---|
| **基盤** | 脳全体の汎用機能 | ベースモデルの汎用言語能力 |
| **制約** | 脳の一部の抑制/損傷 | 大部分のパラメータの固定 |
| **突出** | 特定領域の異常な能力 | 特定タスクへの極端な最適化 |
| **トレードオフ** | 汎用性との引き換え | 汎用性は維持されるが、特化領域以外は変化なし |

### 脳科学との接点

- **アラン・スナイダー博士（シドニー大学）の仮説**: サヴァン症候群の人々は、通常の脳が行う「高次の抽象化」を経由せず、「未加工の詳細なデータ（低次情報）」に直接アクセスできる
- **獲得性サヴァン症候群**: 事故や病気で左脳前頭葉を損傷した人が、突然驚異的な芸術的・数学的能力を発揮する症例が報告されている
- LoRA の「情報の通り道を極端に狭く絞る」構造が、サヴァン的な特化を生む

### トゥルーマン2046 での位置づけ（未定）

orange の解釈を作中に反映する方法として検討中:

- **A.** アンセンサードの区別の直後（「外部からの改造」の一種。LoRA もアンセンサードも人間による改造だが、LoRA は特化・サヴァン化、アンセンサードは脱衣）
- **B.** AI の民族セクション（訓練データが「血統」なら、LoRA は「後天的な才能/障害」）
- **C.** 独立した小セクション（「サヴァン LLM 仮説」として知性の閾値議論に絡める）

## 具体例: ACE-Step（音楽生成ローカル AI）

### 概要

- オープンソースのローカル音楽生成 AI（商用利用可能）
- **ハイブリッド構造**: LM（言語モデル、Qwen3 ベース）が楽曲の設計図を作成 + DiT（Diffusion Transformer）が音声を生成
- RTX 3090 で数分の楽曲を10秒未満で生成可能
- VRAM 4GB でも動作

### 機能

- **Text-to-Music**: プロンプトからフル楽曲生成（ボーカル込み、50言語以上対応）
- **Remix**: 既存音源を別ジャンルに変換
- **Repaint**: 曲の指定区間だけを再生成
- **LoRA**: 特定のジャンル・音色・アーティスト風の「クセ」を追加学習

### LoRA の具体的な使われ方

- ベースモデル（バニラ ACE-Step）は「無難なポップス寄り」のバイアスを持つ
- LoRA で特定ジャンル（メタル、アンビエント等）の音色・構成を注入
- 数十曲程度の学習データで効果が出る（ベースモデルの音楽基礎能力があるため）

### 商用利用のリスク

- モデルのライセンス（MIT 等）= ソフトウェアの利用許可であり、出力物の著作権非侵害を保証しない
- AI が既存曲を学習していた場合、類似した出力を商用利用すると**ユーザーが責任を負う**
- 著作権侵害の2要件: **類似性**（客観的に似ている）+ **依拠性**（元の曲を知っていて参考にした）
- 自衛策: Shazam での事前チェック、YouTube Content ID での非公開テスト
